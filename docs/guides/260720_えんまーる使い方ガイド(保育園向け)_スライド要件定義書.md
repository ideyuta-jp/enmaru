## 1. 目的・背景

えんまーるに登録しようとしている保育園・施設運営者に対し、「サインアップ →
園プロフィール記述(写真アップロード含む) → 募集の作成 → マッチング → チャット
やり取り → 勤務完了・給与 → 保育士評価」までの一連の使い方を、短時間で理解して
もらうための説明スライドを作成する。導入商談・オンボーディング時の説明会・動画・
アプリ内ヘルプ等での利用を想定する。

本資料は「260720_えんまーる使い方ガイド(保育士向け)_スライド要件定義書.md」と対の
資料であり、構成・ドキュメントルールを揃えている。

## 2. ターゲットとゴール

| --- | --- |
| ターゲット | えんまーるへの登録を検討している、または登録直後の保育園・施設の運営担当者(園長・採用担当者等) |
| 視聴後のゴール | 登録から保育士評価までの一連の流れが具体的にイメージでき、迷わず最初の一歩(新規登録・園プロフィール作成)を踏み出せる状態になること。利用料金の仕組みも理解していること |
| 視聴シーン | 導入商談・登録前の説明・登録直後のオンボーディング動画・説明会での投影 |

## 5. 各スライドの詳細要件

各スライド共通で「画面要素(何を見せるか)」「注意事項」を満たすこと。台詞(ナレーション)は
別紙「スライド原稿」を参照。

1. **タイトル**: サービスロゴ、サブタイトル「保育園・施設運営者向け使い方ガイド」。
2. **えんまーるとは**: 「登録 → 園プロフィール → 募集作成 → マッチング → チャット
   → 勤務・完了報告 → 給与 → 評価」のフロー図。保育士向けガイドのフロー図と対称
   になるよう構成する(保育士側は「応募」、保育園側は「募集作成」が起点)。
3. **サインアップ・登録**: ヘッダーの「新規登録」ボタン → 登録区分選択(「保育園・
   施設運営の方」カード)→ 利用規約(`/terms/nursery`)同意チェック → 「登録を完了
   する」ボタン。**保育士向けと画面・処理は共通コンポーネントだが、選ぶカードと
   同意する規約が異なる**点に注意(取り違え防止)。
4. **園プロフィールを入力する**: 「園プロフィール編集」画面。園名・住所(郵便番号
   から自動補完)・電話番号・担当者名(以上必須)、ホームページ・SNSリンク、園の
   特徴タグ(最大3つ)、受け入れ時に大切にしていることタグ(最大3つ)、えんまーる
   に参加した理由・こんな方とご縁があれば嬉しい等の自由記述、公開設定チェック、
   「保存する」ボタンを紹介する。
5. **園の写真をアップロードする**: メイン写真(1枚、16:9推奨、ドラッグ&ズームで
   クロップしてから登録)とサブ写真(最大5枚、クロップなしでそのまま登録)の2種類
   があることを明確に説明する。**プロフィールを一度も保存していない初回訪問時から
   写真アップロードのセクションが表示され、すぐに写真を登録できる**(2026-07時点
   の改善)ことをポジティブに紹介する。対応形式(JPEG/PNG/WebP、各5MBまで)も明記。
6. **募集を作成する**: 「募集管理」画面の「新規作成」ボタン → タイトル・勤務内容
   タグ・勤務日時・時給(任意)・必要な資格・交通費・服装・求める人物像・**応募に
   必要な書類のチェック(履歴書/保育士証/健康診断書/検便結果)**・備考を入力し
   「作成する」。**カレンダーで複数日をまとめて選択すると、その日数分の募集を
   一度に作成できる**効率化ポイントを紹介する。作成済みの募集は「募集を終了する」
   で公開停止でき、**削除機能はなく終了(公開停止)のみである**点も明記する(誤解
   防止)。
7. **応募・マッチングを確認する**: 「応募管理」画面。応募 = 即時マッチング成立
   (先着順・面接なし)であること、**マッチング成立と同時に保育士の本名が開示され
   る**こと(表示名と実名が併記される)を明確に説明する。「新着応募」「その他」の
   区分、「詳細・チャット」ボタンを紹介する。
8. **チャットでやり取りする**: チャット画面(保育士向けと共通のUI)。**チャットは
   業務完了報告から24時間で自動的に利用できなくなる**点を必ず明記する。
9. **勤務当日〜完了報告**: **保育園側には「業務を開始する」ボタンはなく、保育士
   が業務を開始するのを待つ**ことを明確に説明する(誤操作・混乱防止)。業務開始後
   は保育園側からも「業務完了を報告する」ボタン(コメント任意)で報告し、**保育士
   ・保育園の双方が完了報告をして初めて「業務完了」になる**仕組みを説明する。
10. **給与・利用料金について**: 2つの金銭事項を分けて説明する。①**給与**: えん
    まーるアプリ内には給与計算・振込・明細発行の機能は現時点で存在せず、業務完了
    報告後に保育園から保育士へ直接支払う運用である(実装画面のキャプチャは使用し
    ない)。②**えんまーるの利用料金**: 掲載導入サポート料は実証期間(2026年4月1日
    〜2027年3月31日)は無料、以降継続希望の場合は**年額26,400円(税込)**。成功報酬
    型ではない。支払いは請求書に基づく銀行振込。
11. **保育士を評価する**: 「評価を書く」一覧(業務完了案件のみ表示)→評価入力
    フォーム(3項目の5段階評価: 勤務態度/コミュニケーション/保育スキル、コメント
    任意、「またこの保育士に依頼したい」チェック)→「評価を送信する」ボタン。
    **この評価は保育士本人には一切公開されず、事務局のみが今後のマッチング判断の
    参考として管理する**(保育士向け評価が運営確認後に公開されるのとは非対称な
    仕組みである)ことを必ず明記する(誤解防止の最重要ポイント)。
12. **まとめ**: フロー図の再掲、問い合わせ先の案内。

## 6. 実装状況に関する注意事項(誤情報防止のための必読事項)

制作にあたり、以下は現状の実装(2026-07時点)を正しく反映すること。誤って
「アプリ内で給与計算・振込ができる」「保育士に評価が公開される」等と誤解を招く
表現をしないこと。

- **給与精算はアプリ内機能として存在しない。** 業務完了報告(WorkReport)まではアプリ
  内で完結するが、給与の計算・支払いは保育園から保育士への直接のやり取りであり、
  えんまーるは仲介していない。
- **保育士への評価は保育士本人に公開されない。** `/nursery/mypage` の案内文どおり
  「保育士さんには公開されず、事務局のみで安全に管理されます」が正確な仕様。保育士
  向け評価(保育士→保育園)は運営確認後に公開されうるのと非対称なので混同しない。
- **保育園は「業務を開始する」を実行できない。** 業務開始は保育士のみが行い、保育園
  は待機する。業務完了報告は双方が個別に行う。
- マッチングは「応募 = 即時成立・先着順」であり、保育園からの承認や面接のステップ
  は存在しない(保育士側と同じ仕様)。
- チャットは「マッチング成立〜業務完了報告から24時間後」までの時限式(保育士側と
  共通のコンポーネント)。
- **求人(募集)に削除機能はない。** 「募集を終了する」で公開停止するのみで、レコード
  自体を削除するUIは実装されていない。閉じた募集を再度「募集を再開する」ことも可能
  (2026-07時点。将来「一度閉じたら再開不可」という仕様に変わる可能性がある)。
- **写真アップロードは初回訪問時から可能。** プロフィールを一度も保存していなくても、
  写真アップロードのセクションは常に表示される(空のドラフト行が裏側で自動作成される
  ため)。ただし募集の作成には、園名・担当者名を含めて一度保存されたプロフィールが
  必要。
- **キャンセル・不参加時の扱いは未確定**(`docs/requirements.md` で「別途ディスカッ
  ション」と明記された未決事項、保育士向けガイドと同じ注意点)。
- **利用料金**: 掲載導入サポート料は実証期間(2026年4月1日〜2027年3月31日)無料、
  以降継続希望の場合は年額26,400円(税込)。成功報酬型ではない。支払いは請求書に
  基づく銀行振込で、支払期限超過時は年14.5%の遅延損害金が発生し得る。解約は途中
  解約でも既発生料金の返金はなく、期間満了1ヶ月前までに申し出がない場合は自動更新
  となる(`/terms/nursery` 第5・6・8条)。これらの数値は利用規約の改定により変わり
  得るため、動画公開前に最新の規約と一致しているか必ず確認すること。
