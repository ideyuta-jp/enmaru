## 1. 目的・背景

えんまーるに新規登録しようとしている保育士(潜在保育士・保育バディ)に対し、
「サインアップ → プロフィール記述 → 書類アップロード → 応募 → マッチング →
チャットやり取り → 業務完了・給与 → 保育園評価」までの一連の使い方を、短時間で
理解してもらうための説明スライドを作成する。オンボーディング時の説明会・動画・
アプリ内ヘルプ等での利用を想定する。

## 2. ターゲットとゴール

| 項目 | 内容 |
| --- | --- |
| ターゲット | これから、または登録直後の保育士(保育バディ)。保育士資格はあるが現場を離れていた方も多く、Webサービス利用に不慣れな層を含む |
| 視聴後のゴール | 登録から評価までの一連の流れが具体的にイメージでき、迷わず最初の一歩(新規登録)を踏み出せる状態になること |
| 視聴シーン | 登録前の説明・登録直後のオンボーディング動画・説明会での投影 |

## 3. 全体要件

| 項目 | 要件 |

| 想定尺 | 合計 約5分(300〜320秒目安、1スライド15〜35秒) |

## 5. 各スライドの詳細要件

各スライド共通で「画面要素(何を見せるか)」「注意事項」を満たすこと。台詞(ナレーション)は
別紙「スライド原稿」を参照。

1. **タイトル**: サービスロゴ、サブタイトル「保育士(保育バディ)向け使い方ガイド」。
2. **えんまーるとは**: 「登録 → プロフィール → 書類提出 → 応募 → マッチング →
   勤務 → 給与 → 評価」のフロー図。以後のスライドと対応が分かるようにする。
3. **サインアップ・登録**: ヘッダーの「新規登録」ボタン → 登録区分選択(「保育士・
   保育補助希望の方」カード)→ 利用規約同意チェック → 「登録を完了する」ボタン。
   LINE公式アカウントの友だち追加案内があることも触れる。
4. **プロフィール入力**: 「プロフィール編集」画面。表示名・希望エリア・保有資格・
   保育経験・得意分野・自己紹介などの入力項目と、「保存する」ボタン。**一部項目
   (本名・ブランク期間・職務経歴)はマッチング成立後の保育園にのみ公開される**
   点を必ず明記する(誤解防止)。**特に本名については、「マッチングが成立した
   瞬間に相手の保育園へ開示される」という個人情報に関わる重要な事実であるため、
   曖昧にせず明確に伝えること。**
5. **必要書類のアップロード**: 「書類の提出」画面。必須書類(健康診断書・履歴書)
   と任意書類(保育士証・検便結果、募集により必須)、「アップロード」ボタン、
   対応形式(JPEG・PNG・WebP・PDF、10MBまで)、審査ステータス(確認中/認証済み/
   差し戻し)。**応募には必要書類の認証が必須(ドキュメントゲート)**である点を
   明記する。
6. **保育園を探して応募する**: 「保育園一覧」→ 保育園詳細ページの「現在の募集」
   → 「この募集に応募する」ボタン → 応募フォーム(応募メッセージ・LINE連絡許可)
   → 「応募する」ボタン。
7. **マッチング成立**: **応募した時点で即時マッチングが成立し、先着順で他の応募
   者には募集が締め切られる(面接・審査なし)**ことを明確に説明する。マッチング
   成立の通知が届くことも触れる。
8. **チャットでやり取り**: チャット画面(吹き出しUI、メッセージ入力欄、「送信」
   ボタン)。**チャットは業務完了報告から24時間で自動的に利用できなくなる**点を
   必ず明記する。
9. **勤務当日〜完了報告**: マッチング詳細画面の「業務を開始する」ボタン→勤務後
   「業務完了を報告する」ボタン(コメント任意)。**保育士・保育園の双方が完了報
   告をして初めて「業務完了」になる**仕組みを説明する。
10. **給与について**: **重要:えんまーるアプリ内には給与計算・振込・明細発行の
    機能は現時点で存在しない。** 業務完了報告後、勤務した保育園から保育士へ
    「直接」給与が支払われる運用である旨を、概念図(保育園 → 保育士の支払い矢印)
    でシンプルに説明する。支払い時期・方法は保育園ごとに異なるためチャット等で
    確認を促す文言を入れる。実装画面のキャプチャは使用しない(存在しないため)。
11. **保育園を評価する**: 「評価を書く」一覧(業務完了案件のみ表示)→評価入力
    フォーム(4項目の5段階評価: 説明のわかりやすさ/職場の雰囲気/サポート体制/
    業務内容の明確さ、コメント任意、「またこの保育園で働きたい」チェック)→
    「評価を送信する」ボタン。**評価は業務完了後のみ投稿可能で、投稿直後は非公開
    、運営確認後に公開される**仕組みを説明する。
12. **まとめ**: フロー図の再掲、問い合わせ先の案内。

## 6. 実装状況に関する注意事項(誤情報防止のための必読事項)

制作にあたり、以下は現状の実装(2026-07時点)を正しく反映すること。誤って
「アプリ内で給与計算・振込ができる」等と誤解を招く表現をしないこと。

- **給与精算はアプリ内機能として存在しない。** 業務完了報告(WorkReport)まではアプリ
  内で完結するが、給与の計算・支払いは保育園から保育士への直接のやり取りであり、
  えんまーるは仲介していない。スライドではこの点を正直に、かつネガティブな印象
  にならないよう「安心して直接やり取りできる関係性ができている」という前向きな
  トーンで説明する。
- マッチングは「応募 = 即時成立・先着順」であり、園からの承認や面接のステップは
  存在しない。
- チャットは「マッチング成立〜業務完了報告から24時間後」までの時限式。
- 評価は「業務完了(COMPLETED)後のみ」投稿可能で、投稿後は非公開 → 運営確認後に
  公開、という2段階の仕組み。
- プロフィールは「公開」「マッチング相手のみ公開」「非公開(本人のみ)」の3段階の
  公開範囲がある。
- **本名の開示タイミング**: マッチングが成立すると同時に、保育士の本名が相手保育園に
  開示される(応募 = マッチング成立のため、応募した瞬間に開示される)。事前の同意
  ステップは別途存在しない。
- **キャンセル・不参加時の扱いは未確定**(`docs/requirements.md` で「別途ディスカッ
  ション」と明記された未決事項)。マッチング成立後に保育士が参加できなくなった場合の
  正式なルール・ペナルティはサービスとしてまだ定まっていない。本編スライドでは断定的
  な案内をせず、FAQで「まずチャットで保育園に連絡し、あわせて運営にも連絡する」という
  暫定的な行動指針にとどめる(正式なキャンセルポリシーが決まり次第、本資料も更新する)。
- **複数の募集への同時応募は可能**(運営確認済みの業務ルール、2026-07-20時点)。
  - 同じ保育園の別の募集: 勤務時間が重複していなければ、同じ日付でも応募できる。
  - 異なる保育園の募集: 勤務日が異なる場合に応募できる(同日は不可)。
