## 0. 制作方針(撮影前に必ず確認)

- **形式**: PC(デスクトップ)ブラウザの画面収録+ナレーション音声+テロップ合成による解説動画。出演者による実写パートは必須ではない(タイトル・フロー図カットのみモーショングラフィックスで作成)。**保育士向け台本はスマホ画面が基本だったのに対し、本台本はPC画面が基本**(保育園はPC利用が中心のため)。
- **構成**: 本編(カット1〜12、約5分、ナレーションあり)+ FAQエンドカード(カット13〜21、視聴後の参考ページ。導入以外はテロップ主体・ナレーション最小限)。
- **収録アカウント**: 必ず撮影・収録専用の**テスト(ダミー)アカウント**を用いる。実在の保育園・保育士の個人情報(園名・住所・電話番号・担当者名・保育士の氏名等)は一切映さない。
- **写真素材**: 園プロフィールの写真アップロード(カット5)は、権利処理済みのフリー素材またはダミーの保育室イメージ画像を使用する。実在の園児が写っている写真は使用しない。
- **料金情報の正確性**: カット10・カット21で扱う利用料金(掲載導入サポート料)は、収録直前に `/terms/nursery` の最新の記載内容と一致しているか必ず確認する。利用規約が改定されている場合は金額・条件を台本ごと更新してから収録する。

## 3. 本編カット台本(カット1〜12、目安 約5分)

### カット1:タイトル(00:00–00:15)

- **画面/操作**: 実機操作なし。タイトルカード(サービスロゴ+サブタイトル)を静止表示。編集ソフトで作成。
- **テロップ**: 「えんまーる|保育園・施設運営者向け 使い方ガイド」
- **ナレーション**: 「こんにちは。ここでは、保育士と保育園をつなぐマッチングサービス『えんまーる』の使い方を、登録から保育士を評価するまでの流れに沿って、約5分でご紹介します。」
- **収録メモ**: BGMをこのカットからフェードイン。軽い開始SEを1つ。

### カット2:えんまーるとは(00:15–00:40)

- **画面/操作**: 実機操作なし。フロー図(登録→園プロフィール→写真登録→募集作成→マッチング→チャット→勤務・完了報告→給与→評価の9ステップ)をアイコン+矢印でアニメーション表示。モーショングラフィックスで作成し、以降のカットでも使い回す。
- **テロップ**: 各ステップ名を順番にハイライト表示
- **ナレーション**: 「えんまーるは、保育士資格をお持ちで今は現場を離れている『潜在保育士』の方と、人手を必要としている保育園を、段階的につなぐサービスです。園が募集を出し、保育士が応募すると同時にマッチングが成立し、実際に働いてもらったあとで、お互いに評価し合う仕組みになっています。これから、その一つひとつのステップを見ていきましょう。」
- **収録メモ**: このフロー図データはカット12でも再利用するため、先に完成させておく。

### カット3:サインアップ・登録(00:40–01:10)

- **画面/操作(実機収録・テストアカウントで実演)**:
  1. トップページを表示 → ヘッダーの「新規登録」をクリック
  2. 登録区分選択画面で「保育園・施設運営の方」カードをクリック
  3. 利用規約(保育園向け)画面までスクロールし、「利用規約に同意します」にチェック
  4. 「登録を完了する」をクリック → 完了、`/nursery/mypage` へ遷移
- **テロップ**: 「①新規登録 → ②『保育園・施設運営の方』を選択 → ③規約に同意 → ④登録を完了する」
- **ナレーション**: 「まずは新規登録です。トップページの『新規登録』ボタンから進み、『保育園・施設運営の方』を選択します。利用規約に同意して『登録を完了する』を押せば、登録は完了です。」
- **収録メモ**: 保育士向け動画と同じ登録画面を使うが、選ぶカードが異なる点を誤って逆に撮らないよう注意する。

### カット4:園プロフィールを入力する(01:10–01:45)

- **画面/操作(実機収録)**:
  1. マイページ→「園プロフィール編集」をクリック
  2. 園名・郵便番号(自動で住所補完される様子)・電話番号・担当者名を入力
  3. ホームページ・SNSリンク、園の特徴タグ、受け入れ時に大切にしていることタグ、メッセージ欄をスクロールしながら映す(すでに入力済みのダミーデータを見せる形でよい)
  4. 「保存する」をクリック → 保存完了のスナックバー表示
- **テロップ**: 「園名・住所・担当者名は必須。その他の項目は任意」
- **ナレーション**: 「次に、『園プロフィール編集』画面で、園の基本情報を入力しましょう。園名や住所、担当者名のほか、ホームページやSNSのリンク、園の特徴、受け入れの際に大切にしていることなどを入力できます。入力できたら『保存する』を押します。」
- **収録メモ**: 住所・電話番号・担当者名は必ずダミーデータ。郵便番号入力後に住所が自動補完される様子は分かりやすいので、その瞬間を必ず収録する。

### カット5:園の写真をアップロードする(01:45–02:15)

- **画面/操作(実機収録)**:
  1. 同じ「園プロフィール編集」画面内、メイン写真セクションで画像を選択 → クロップ編集モーダルが開く → ドラッグで位置調整、スライダーでズーム → 「確定」をクリック
  2. サブ写真セクションで複数枚(2〜3枚)を選択してアップロード → サムネイルが並ぶ様子を映す
- **テロップ**: 「メイン写真は1枚(クロップあり)、サブ写真は最大5枚(クロップなし)」
- **ナレーション**: 「プロフィールには、園の様子が伝わる写真も登録できます。メイン写真は1枚、ドラッグとズームで見せたい部分を調整してから登録します。サブ写真は最大5枚まで、そのまま登録できます。プロフィールを保存する前でも写真だけ先に登録できるので、まずは気軽にアップロードしてみてください。」
- **収録メモ**: 使用する写真は権利処理済みのダミー素材のみ。実在の園児が写った写真は絶対に使用しない。クロップ編集のドラッグ・ズーム操作はややゆっくり行い、視聴者が操作方法を追えるようにする。

### カット6:募集を作成する(02:15–02:50)

- **画面/操作(実機収録)**:
  1. 「募集管理」画面を表示 →「新規作成」をクリック
  2. タイトル(サジェストチップから選択可)、勤務内容タグ、勤務日時(カレンダーで複数日選択できる様子)、時給(任意)を入力
  3. 「応募に必要な書類」のチェック欄(履歴書・保育士証・健康診断書・検便結果)を映す
  4. 「作成する」をクリック → 完了トースト表示
- **テロップ**: 「複数日をまとめて選ぶと、その日数分の募集を一度に作成できます」
- **ナレーション**: 「保育士に来てもらうには、『募集管理』画面から募集を作成します。タイトルや勤務内容、日時、時給、必要な資格などを入力し、応募に必要な書類にチェックを入れて『作成する』を押します。カレンダーで複数の日をまとめて選ぶと、その日数分の募集を一度に作成できて便利です。」
- **収録メモ**: 「募集を終了する」ボタンがあることには軽く触れる程度でよい(詳細はFAQカット16で扱う)。削除機能はないため、誤って「削除できます」という説明にならないよう注意。

### カット7:応募・マッチングを確認する(02:50–03:20)

- **画面/操作**: 事前にテスト保育士アカウントで応募・マッチング済みにしておいたデータを「応募管理」画面で表示。「新着応募」のカードに保育士の表示名と実名が並記されている様子を映す。
- **テロップ**: 「応募 = 即時マッチング成立(先着順・面接なし)」
- **ナレーション**: 「保育士が応募すると、その時点で即座にマッチングが成立します。面接や審査はありません。『応募管理』画面で、応募してくれた保育士の情報を確認できます。マッチングが成立すると、保育士の本名もあわせて表示されます。」
- **収録メモ**: 実名部分もダミーデータであることを再確認すること。

### カット8:チャットでやり取りする(03:20–03:45)

- **画面/操作**: マッチング詳細画面のチャットを表示。事前に送受信しておいたダミーのやり取り(吹き出し)をスクロールで見せ、「メッセージを入力」欄に軽くクリックして「送信」ボタンを押す動作を1回撮る。
- **テロップ**: 「チャットは業務完了報告から24時間で自動終了」
- **ナレーション**: 「マッチングが成立したら、保育士とチャットで直接やり取りができます。当日の持ち物や集合時間など、事前に確認したいことを伝えましょう。チャットは業務完了から24時間で自動的に終了します。」
- **収録メモ**: チャット文面は当たり障りのないダミー内容にする(持ち物確認など)。

### カット9:勤務当日〜完了報告(03:45–04:15)

- **画面/操作**:
  1. マッチング詳細画面で「保育士の業務開始を待っています」というキャプション表示のみの状態を映す(保育園側にはボタンがないことを見せる)
  2. (別テイクで)保育士が業務を開始した後の状態に切り替え、「業務完了を報告する」をクリック→コメント欄(任意)に一言入力→送信
- **テロップ**: 「業務の開始は保育士側の操作。保育園は完了報告のみ」
- **ナレーション**: 「勤務当日、業務の開始は保育士側の操作で行われるため、保育園側で押すボタンはありません。保育士が業務を開始したら、勤務終了後に園側も『業務完了を報告する』から報告してください。保育士・保育園の双方が報告して初めて『業務完了』となります。」
- **収録メモ**: 「開始待ち」の状態と「完了報告」の状態は同日に連続して撮れないため、2テイクに分けて収録し編集で繋ぐ。

### カット10:給与・利用料金について(04:15–04:40)

- **画面/操作**: 実機操作なし。①「保育園 → 保育士」への支払いを表す矢印イラスト、②シンプルな料金表(実証期間:無料/以降:年額26,400円税込)のモーショングラフィックスを表示(アプリ画面は使用しない)。
- **テロップ**: 「給与はアプリ内では扱いません。保育園から直接お支払い」「利用料金:実証期間中は無料 / 以降は年額26,400円(税込)」
- **ナレーション**: 「給与は、えんまーるのアプリの中では計算・振込を行わず、業務完了後に保育園から保育士へ直接お支払いいただく運用です。また、えんまーるの利用料金は、実証期間中の2026年4月から2027年3月までは無料、それ以降も継続してご利用いただく場合は年額26,400円(税込)となります。」
- **収録メモ**: 給与精算は実装されていない機能のため実機画面を使用しない。料金の金額・期間は収録直前に `/terms/nursery` の最新版と必ず突き合わせる。

### カット11:保育士を評価する(04:40–05:10)

- **画面/操作**: 業務完了(COMPLETED)状態のテストデータで「評価を書く」一覧→対象案件をクリック→評価フォームで5段階評価(星をクリック)→コメント欄にダミー文面→「またこの保育士に依頼したい」にチェック→「評価を送信する」をクリック
- **テロップ**: 「この評価は保育士本人には公開されません(事務局のみで管理)」
- **ナレーション**: 「業務が完了すると、『評価を書く』からその保育士を評価できます。勤務態度やコミュニケーションなど3つの項目を5段階で評価し、感想や『また依頼したいか』を入力して送信します。この評価は保育士本人には公開されず、事務局が今後のマッチングの参考として管理する仕組みです。」
- **収録メモ**: 「保育士には公開されない」という点は誤解が生じやすい重要情報のため、テロップとナレーションの両方で必ず明示する。

## 5. 収録・編集時の注意事項(必読)

- **個人情報**: 実在の園・保育士の名称・住所・電話番号・担当者名・保育士の氏名を映さない。テスト用ダミーデータのみを使用する。誤って実データが映り込んだ場合は必ず編集でぼかし/カットする。
